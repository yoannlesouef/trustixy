# Trustixy

**AI Trust & Compliance, by design.**

Trustixy is a SaaS platform that helps companies comply with the **EU AI Act** — not just once, but continuously. It provides a structured, versioned registry of AI systems, tracks every classification over time, alerts users when regulatory changes require re-assessment, and enables prescriber co-signature to give compliance documents genuine professional standing.

---

## The Problem

The EU AI Act requires companies to:
1. **Register** their AI systems
2. **Classify** their risk level (minimal / limited / high / unacceptable)
3. **Produce compliance documentation**
4. **Maintain an audit trail** — and keep it current as the regulation evolves

Most teams solve this with a one-time PDF or a spreadsheet. That breaks the moment the regulation updates or a system changes. And a document generated by a generic LLM carries no professional weight with an auditor.

---

## What Trustixy Provides

| Capability | What it means |
|---|---|
| **Structured registry** | Every AI system in one place, with compliance status, versioned history, never deleted |
| **Confidence-scored classification** | LLM classification + confidence indicator; low-confidence cases flagged for expert review |
| **Obligations checklist** | Actionable tasks per system, trackable to completion — not just a document section |
| **Versioned classification history** | Every assessment stored: date, regulatory version, confidence, trigger |
| **Continuous compliance status** | Per-system badge (Compliant / Needs Review / At Risk) always up to date |
| **Regulatory change alerts** | Automatic notifications when EU AI Act updates affect registered systems |
| **Prescriber co-signature** | Partners review and co-sign documents — turning AI output into professionally endorsed compliance |
| **Mandatory disclaimer** | Every document carries an explicit legal scope disclaimer — protecting users and Trustixy |
| **Immutable audit trail** | Every action logged, exportable for regulatory submissions |

---

## Human Validation Flow

1. User runs a classification — LLM outputs risk level + confidence score
2. If confidence < 70%: expert review warning shown, co-signature recommended
3. User generates compliance document — disclaimer auto-embedded
4. User requests co-signature from prescriber partner
5. Prescriber reviews classification and document in the partner portal, then signs or requests changes
6. Co-signed document displays verified badge + prescriber name and firm

## Go-to-Market Strategy

Trustixy uses a **B2B2B prescriber model**. Prescribers distribute Trustixy to their SME clients and co-sign compliance documents — creating a recurring billable service for the prescriber and a professionally validated document for the end user. The prescriber owns the client relationship; Trustixy is the infrastructure.

### Three prescriber revenue models

| Model | For | Trustixy cost | Prescriber revenue |
|---|---|---|---|
| **Per-act co-signature** | Accounting firms, consultants | €25 commission/act | Prescriber sets their own fee (€150–500); direct Stripe payout |
| **Federation license** | Professional federations, trade associations | €3 000–10 000/year | Distribute Pro access to all members under one contract |
| **Integrator license** | IT integrators, ESN/SSII | €299–599/month | Bundle compliance into project deliverables at their own price |

Full details: [`/specs/prescriber-partnership.md`](specs/prescriber-partnership.md)

## Monetization

**For organizations:**
* **Free:** up to 3 AI systems, unsigned documents with disclaimer
* **Pro:** unlimited systems, full alert history, co-signature requests, obligation tracking
* **Enterprise:** org compliance report, API access, advanced audit export, SLA

**For prescribers:**
* Per-act billing via Stripe Connect — no subscription required
* Federation license — flat annual fee, members auto-provisioned on Pro
* Integrator license — monthly subscription with API access and white-labeled reports
