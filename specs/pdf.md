# PDF Generation

## Approach

Server-side rendering via `@react-pdf/renderer`. PDFs are generated on-demand and stored in Supabase Storage. No headless browser (Puppeteer) — too heavy for a serverless environment.

```bash
npm install @react-pdf/renderer
```

---

## Generation Flow

1. User clicks [Generate document] on the Agent page
2. Server action calls `generateDocument(agentId, classificationId)`
3. LLM generates document content (see `specs/llm-prompts.md` — Prompt 3)
4. Server renders PDF from content using `@react-pdf/renderer`
5. PDF uploaded to Supabase Storage: `documents/[org-id]/[doc-id].pdf`
6. `documents` row inserted: `content`, `version`, `disclaimer_version`, `signature_status = unsigned`, `is_current = true`
7. Previous document's `is_current` set to `false`
8. Signed URL returned to client (1-hour expiry)
9. Browser triggers download

## PDF Template Structure

- A4 format
- Margins: 40pt all sides
- Font: Inter (loaded via `@react-pdf/font`)
- Primary color: #0F172A
- Accent: #3B82F6

### Disclaimer block (mandatory, non-removable)

Always appears on the last page before the footer. Exact text:

> *"This document was generated by Trustixy using AI assistance and operational data from the Trustixy SDK. It is provided to support your compliance process and does not constitute legal advice. Trustixy cannot be held responsible for regulatory decisions made solely on the basis of this document. For legally binding compliance, we recommend co-signature by a qualified prescriber partner or review by a qualified legal professional."*

On co-signed documents, the disclaimer is still present but a second block is added above it:

> *"This document has been professionally reviewed and co-signed by [Name], [Firm] on [date]. The co-signer has confirmed the classification and obligations are appropriate to their professional judgment within the scope defined in the Trustixy Partner Agreement."*

## Storage

Bucket: `documents` (private)

Path: `documents/[organization-id]/[document-id].pdf`

Access: via Supabase signed URL with 1-hour expiry. Generated server-side on each download request — never expose a long-lived public URL.

Co-signature review link: 24-hour signed URL sent to prescriber in the notification email.

---

## Versioning

When a document is regenerated (reclassification or manual regeneration):
- New `documents` row created with `version = previous_version + 1`, `is_current = true`
- Previous document: `is_current = false`
- Previous document remains accessible via "Show history" toggle on agent page
- If previous document was co-signed, its signature status is preserved — it remains a valid historical record
- Verification page for the old doc-id shows status `Superseded` + links to current version

## Error Handling

| Failure | Response |
|---|---|
| LLM call fails during generation | Show error: "Document generation failed. Try again." — do not create a partial document row |
| PDF render fails | Log error, return 500, show: "PDF export failed. Try again or contact support." |
| Storage upload fails | Retry once; if still failing, return the PDF as a stream directly (don't store) |
| Verification page: doc not found | Show: "Document not found or has been revoked." — no 404 |
